# 1. Purpose

This policy establishes the change management framework for the Blowbits Solutions LLP GST Suvidha Provider (GSP) platform.

The objective of this policy is to ensure that changes to applications, infrastructure, configurations and operational processes are planned, reviewed, approved and implemented in a controlled manner to minimise operational and security risks.

# 2. Scope

This policy applies to application software, infrastructure components, system configurations, security controls and operational processes associated with the Blowbits Solutions LLP GST Suvidha Provider (GSP) platform.

Change management coverage includes:

- Application and API changes.
- Infrastructure, operating system and database changes.
- Network, firewall and security configuration changes.
- Deployment, runtime and third-party component changes.
- Emergency changes required to address urgent operational or security issues.
- Changes affecting GSTN integration, ASP access or production operations.

Supporting procedures, technical standards and implementation evidence are maintained separately.

# 3. Change Management Framework

Changes affecting the GSP platform are managed through structured governance processes.

The framework is established to:

- Ensure proposed changes are documented and evaluated.
- Assess the operational and security impact of changes.
- Support appropriate review and approval prior to implementation.
- Minimise disruption to production services.
- Maintain records of implemented changes.

# 4. Roles and Responsibilities

Change management responsibilities shall be assigned to authorised internal roles defined in the Roles, Responsibilities & Access Matrix.

| Role | Change Management Responsibility |
| ---- | -------------------------------- |
| **Platform Operations Lead** | Coordinate technical impact assessment, implementation planning, deployment, rollback planning and post-change validation. |
| **Information Security Lead** | Review security impact for changes affecting access, configurations, vulnerabilities, security controls or sensitive information. |
| **GSP Governance Lead** | Approve significant production changes, emergency changes, exceptions and changes with material operational, security or regulatory impact. |
| **GSP Coordination Lead** | Coordinate GSTN or authorised external communication where changes affect GSP operations or external coordination. |

# 5. Change Types

Changes shall be classified based on operational and security impact.

Change types may include:

- Standard or routine changes.
- Normal production changes.
- Security changes.
- Emergency changes.
- Significant changes affecting availability, security, data integrity, regulatory commitments or external communication.

# 6. Change Management Principles

The GSP platform operates according to the following principles:

- Changes are initiated through controlled operational processes.
- Security and operational impacts are considered before implementation.
- Changes are reviewed and approved by authorised personnel.
- Production changes are validated after implementation.
- Significant changes include implementation planning, rollback or recovery consideration and post-implementation review where appropriate.
- Emergency changes are documented and reviewed after implementation.
- Change activities are documented and retained.
- Significant changes are reviewed following implementation where appropriate.

# 7. Change Management Process

Change management activities are performed through structured operational procedures.

The operational framework includes:

- Change request initiation.
- Impact assessment.
- Review and approval.
- Implementation planning.
- Rollback or recovery planning where applicable.
- Validation and deployment.
- Documentation and closure.

# 8. Review

This policy shall be reviewed annually or whenever significant changes occur to the GSP platform, change management process or applicable operational requirements.

# 9. Compliance

Compliance with this policy is supported through documented procedures, operational practices, change records, approval records, validation evidence, emergency change review records and implementation evidence.

Detailed operational processes and technical controls are maintained separately to support the effective implementation of this policy.
