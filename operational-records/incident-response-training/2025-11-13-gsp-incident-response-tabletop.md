# 1. Purpose

This document records the GSP incident response training and tabletop exercise conducted for the Blowbits Solutions LLP GST Suvidha Provider (GSP) incident response team.

The objective of the training was to ensure that designated personnel understand their responsibilities for identifying, classifying, escalating, containing, investigating and closing security incidents affecting the GSP platform, GSTN integration, taxpayer information or supporting infrastructure.

# 4. Topics Covered

The following topics were reviewed and discussed:

- Security incident identification and reporting process.
- Incident severity classification for Critical, High, Medium and Low incidents.
- Target response timelines based on incident severity.
- Internal escalation matrix and use of the Contact Directory.
- GSTN and CERT-In notification requirements.
- Incident containment and recovery procedures.
- Evidence collection and log preservation.
- Root cause analysis approach.
- Corrective and preventive action tracking.
- Incident closure and post-incident review process.

# 5. Tabletop Exercise

## 5.1 Scenario

A simulated scenario was discussed in which unusual activity was detected on a production GSP API server. Participants walked through the incident response lifecycle from detection through closure.

## 5.2 Scenario Walkthrough

| Stage                   | Discussion Summary                                                                                                                                                                                                  |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Detection               | Monitoring systems generated alerts indicating abnormal API traffic. The operations team validated that the activity required investigation.                                                                        |
| Severity Classification | The initial severity was classified as High based on potential production impact.                                                                                                                                   |
| Escalation              | The escalation path reviewed was Platform Operations Team, Information Security Lead, GSP Coordination Lead, GSP Governance Lead and external notification where required.                                          |
| Containment             | Participants discussed restricting suspicious traffic, preserving affected system logs, verifying application integrity, isolating affected components where required and increasing monitoring on related systems. |
| External Notification   | The team reviewed decision criteria for notifying GSTN and CERT-In in accordance with organisational procedures and applicable regulatory requirements.                                                             |
| Recovery                | Participants reviewed service integrity validation, restoration of affected services, continuous post-recovery monitoring and verification of incident resolution.                                                  |
| Closure                 | The closure process reviewed root cause analysis, documentation of findings, corrective and preventive actions, management review and formal incident closure.                                                      |
