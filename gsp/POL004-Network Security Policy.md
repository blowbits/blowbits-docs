# 1. Purpose

This policy establishes the network security framework for the Blowbits Solutions LLP GST Suvidha Provider (GSP) platform.

The objective of this policy is to protect network infrastructure, communication channels and network-accessible services from unauthorized access while supporting secure, reliable and controlled connectivity between applications, infrastructure and external services.

# 2. Scope

This policy applies to network infrastructure, cloud network controls, firewall or security group rules, communication links, security gateways, routing components, public endpoints and network services associated with the operation of the Blowbits Solutions LLP GSP platform.

Supporting procedures, technical standards, network configuration records, review records and implementation evidence are maintained separately.

# 3. Roles and Accountability

Network security responsibilities are assigned through defined internal roles and operational processes.

The Roles, Responsibilities & Access Matrix defines role-level accountability, authority and authorised access for internal roles.

Network security governance is supported by the following role responsibilities:

- The Platform Operations Lead implements approved network configuration, routing, firewall, security group and connectivity changes.
- The Information Security Lead reviews network security posture, network-related security observations and network control effectiveness.
- The GSP Governance Lead approves significant network security exceptions or high-risk network changes where required.

# 4. Network Security Framework

Network security controls are implemented to regulate communication between authorised users, applications, infrastructure components and external entities.

The framework is established to:

- Protect network boundaries from unauthorized access.
- Regulate inbound and outbound network communications.
- Restrict network access to authorised services, ports, protocols, sources and destinations.
- Support secure communication between interconnected systems.
- Maintain logical separation between public-facing services and internal processing environments.
- Maintain the integrity and availability of network services.
- Support monitoring and review of network security controls.

# 5. Network Security Principles

The GSP platform operates according to the following principles:

- Network communication is permitted only through authorised pathways.
- Inbound and outbound traffic is restricted to required operational and security needs.
- Public endpoints are limited to required services.
- Internal services shall not be directly exposed to the public internet unless approved through a controlled process.
- Access to network services is restricted according to operational requirements and least privilege.
- Network boundaries are protected through firewall, security group, gateway or equivalent controls.
- Public-facing services are logically separated from internal processing environments.
- TLS or equivalent secure transport shall be used for external and sensitive communications where applicable.
- Network activities are monitored to support operational visibility and security.
- Network security controls are periodically reviewed to maintain their effectiveness.

# 6. Network Operations

Network infrastructure is managed through controlled operational processes.

The operational framework includes:

- Configuration of network security controls.
- Management of authorised communication pathways.
- Review of firewall, security group and network access rules.
- Monitoring of network availability, performance and security-relevant events.
- Controlled implementation of network changes through change management.
- Investigation and remediation of significant network security observations.

# 7. Exceptions

Exceptions to network security requirements shall be reviewed, approved and documented by the GSP Governance Lead where applicable.

Approved exceptions shall include a business justification, compensating controls where required and a review or expiry condition where practical.

# 8. Compliance

Compliance with this policy is supported through documented procedures, operational practices, network configuration records, firewall or security group review records, monitoring evidence, change records, exception records and implementation evidence.

Detailed operational processes and technical controls are maintained separately to support the effective implementation of this policy.

# 9. Review

This policy shall be reviewed annually or whenever significant operational, organisational, technical, regulatory or network architecture changes occur.
