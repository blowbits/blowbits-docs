# 1. Purpose

This policy establishes the infrastructure security framework for the Blowbits Solutions LLP GST Suvidha Provider (GSP) platform.

The objective of this policy is to ensure that computing infrastructure supporting GSP services is deployed, operated, monitored and maintained in a secure manner to protect information assets and support reliable service delivery.

# 2. Scope

This policy applies to servers, virtual machines, cloud infrastructure, operating systems, storage resources, runtime environments, administrative tooling, infrastructure configurations and supporting infrastructure associated with the Blowbits Solutions LLP GSP platform.

Supporting procedures, technical standards, configuration records, monitoring records, review records and implementation evidence are maintained separately.

# 3. Roles and Accountability

Infrastructure security responsibilities are assigned through defined internal roles and operational processes.

The Roles, Responsibilities & Access Matrix defines role-level accountability, authority and authorised access for internal roles.

Infrastructure security governance is supported by the following role responsibilities:

- The Platform Operations Lead implements and maintains approved infrastructure resources, configurations, monitoring, maintenance and operational recovery controls.
- The Information Security Lead reviews infrastructure security posture, vulnerabilities, security observations and infrastructure security control effectiveness.
- The GSP Governance Lead approves significant infrastructure security exceptions, privileged infrastructure access and significant risk decisions where required.

# 4. Infrastructure Security Framework

Infrastructure security controls are implemented to protect production environments throughout their operational lifecycle.

The framework is established to:

- Protect infrastructure against unauthorized access.
- Maintain secure operating environments.
- Support reliable platform operation.
- Protect infrastructure configurations from unauthorized modification.
- Support secure administration and maintenance activities.
- Support infrastructure monitoring, resilience, backup and recovery readiness.
- Support controlled review and remediation of infrastructure security observations.

# 5. Infrastructure Security Principles

The GSP platform operates according to the following principles:

- Production infrastructure is configured using approved security baselines.
- Infrastructure components are administered only by authorised personnel.
- Administrative access follows access control, least privilege and privileged access approval requirements.
- Infrastructure resources are protected using appropriate security controls.
- Infrastructure configurations are protected from unauthorised modification.
- Administrative activities are performed through controlled management processes.
- Infrastructure changes are implemented through change management.
- Security updates and patches are applied through approved operational processes.
- Infrastructure health, availability and security-relevant events are monitored.
- Infrastructure security controls are periodically reviewed.
- Resilience is supported through availability, backup and recovery controls.

# 6. Infrastructure Operations

Infrastructure components are managed through structured operational processes.

The operational framework includes:

- Deployment of approved infrastructure resources.
- Configuration management and secure baseline maintenance.
- Secure administration.
- Operational and security monitoring.
- Patch and maintenance activities.
- Backup, recovery and resilience support.
- Controlled infrastructure modifications through change management.
- Review and remediation of infrastructure security observations.

# 7. Exceptions

Exceptions to infrastructure security requirements shall be reviewed, approved and documented by the GSP Governance Lead where applicable.

Approved exceptions shall include a business justification, compensating controls where required and a review or expiry condition where practical.

# 8. Compliance

Compliance with this policy is supported through documented procedures, operational practices, infrastructure configuration records, monitoring records, patch or maintenance records, access review records, change records, exception records, recovery evidence and implementation evidence.

Detailed operational processes and technical controls are maintained separately to support the effective implementation of this policy.

# 9. Review

This policy shall be reviewed annually or whenever significant operational, organisational, technical, regulatory or infrastructure architecture changes occur.
